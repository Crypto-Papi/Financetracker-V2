Keel

Information Security Policy

Effective Date: December 15, 2024
Version: 1.0

1. Purpose and Objectives

This Information Security Policy (ISP) establishes the framework for protecting the confidentiality, integrity, and availability of information assets at Keel. Our objectives are to:

  • Protect consumer financial data from unauthorized access, disclosure, or misuse
  • Ensure compliance with applicable laws, regulations, and contractual obligations
  • Maintain the trust of our users by implementing industry-standard security controls
  • Establish clear accountability for information security responsibilities
  • Enable secure business operations while managing information security risks

2. Scope

This policy applies to:

  • All information systems used to process, store, or transmit consumer data
  • All third-party services integrated with Keel (Plaid, Stripe, Firebase, Vercel)
  • All personnel with access to company systems (currently sole proprietor)
  • All devices used to access production systems

3. Roles and Accountability

3.1 Information Security Owner

Antonio Monroy, Founder - Responsible for:

  • Overall information security strategy and policy approval
  • Ensuring adequate security controls are implemented
  • Reviewing and responding to security incidents
  • Annual review and update of security policies
  • Compliance with third-party security requirements (Plaid, Stripe)

4. Security Controls

4.1 Access Control

  • Multi-factor authentication (MFA) required for all administrative access
  • Principle of least privilege applied to all system access
  • User authentication required for all application access (Firebase Auth)
  • Regular review of access permissions

4.2 Data Protection

  • All data encrypted in transit using TLS 1.2 or higher
  • All data encrypted at rest using AES-256 encryption
  • Sensitive credentials stored securely (environment variables, not in code)
  • No storage of bank login credentials (handled by Plaid)

4.3 Infrastructure Security

  • Use of managed cloud services with SOC 2 Type II certification (Google Cloud, Vercel)
  • Serverless architecture eliminates server management vulnerabilities
  • Automatic security patching by platform providers
  • HTTPS enforced on all endpoints

5. Third-Party Security

All third-party services are evaluated for security compliance:

  • Plaid - SOC 2 Type II certified, handles bank authentication
  • Stripe - PCI DSS Level 1 certified, handles payment processing
  • Firebase/Google Cloud - SOC 2, ISO 27001 certified, hosts application data
  • Vercel - SOC 2 Type II certified, hosts application frontend

6. Incident Response

In the event of a security incident:

  • Identification: Monitor for and identify potential security incidents
  • Containment: Immediately isolate affected systems
  • Notification: Notify affected users and relevant parties (Plaid, authorities) as required
  • Recovery: Restore systems and data from secure backups
  • Lessons Learned: Document incident and implement preventive measures

7. Acceptable Use

  • Production systems accessed only from secure, updated devices
  • No sharing of credentials or access tokens
  • Secure disposal of any printed sensitive information
  • Immediate reporting of any suspected security issues

8. Policy Review

This policy shall be:

  • Reviewed and updated annually, at minimum
  • Updated when significant changes occur to systems or processes
  • Updated when required by regulatory or contractual changes
